exeptions ( FormatException και το OverflowException) με χρηση try catch

Στη C#, το FormatException και το OverflowException είναι δύο τύποι εξαιρέσεων που μπορούν να συμβούν κατά τη διάρκεια της μετατροπής δεδομένων ή της εκτέλεσης αριθμητικών λειτουργιών.
FormatException



Ένα FormatException 


εκτοξεύεται όταν η μορφή ενός δεδομένου δεν συμφωνεί με την αναμενόμενη μορφή ή όταν μια προσπάθεια μετατροπής ενός string σε έναν διαφορετικό τύπο δεδομένων αποτυγχάνει. 
Για παράδειγμα, αν χρησιμοποιήσετε τη μέθοδο int.Parse για να μετατρέψετε το string "abc" σε έναν ακέραιο, 
θα συναντήσετε ένα FormatException επειδή το string δεν μπορεί να αναλυθεί σε έναν ακέραιο.




OverflowException



Ένα OverflowException εκτοξεύεται όταν μια αριθμητική λειτουργία σε μια μεταβλητή παράγει μια τιμή που είναι έξω από τα όρια του τύπου δεδομένων που μπορεί να αποθηκεύσει. 
Για παράδειγμα, αν προσπαθήσετε να αποδώσετε την τιμή 256 σε μια μεταβλητή τύπου byte, η οποία μπορεί να αποθηκεύσει τιμές από 0 έως 255, θα συναντήσετε ένα OverflowException.



Και οι δύο εξαιρέσεις πρέπει να διαχειριστούν κατάλληλα για να αποφευχθεί το κρασάρισμα του προγράμματος. Ο χειρισμός σφαλμάτων μέσω των μπλοκ try-catch είναι μια κοινή 
πρακτική στην προγραμματισμό C# για να διασφαλιστεί ότι το πρόγραμμα μπορεί να συνεχίσει να τρέχει ή να ενημερώσει τον χρήστη με κατάλληλο τρόπο όταν συναντήσει τέτοιου είδους λάθη.



using System;

class Program {
    static void Main() {
        Console.WriteLine("Παρακαλώ εισάγετε έναν αριθμό:");
        string input = Console.ReadLine();

        try {
            int number = int.Parse(input);
            Console.WriteLine($"Ο αριθμός που εισάγατε είναι: {number}");
        } catch (FormatException) {
            Console.WriteLine("Το κείμενο που εισάγατε δεν είναι έγκυρος αριθμός.");
        } catch (OverflowException) {
            Console.WriteLine("Ο αριθμός είναι πολύ μεγάλος ή πολύ μικρός για να χωρέσει σε ένα Int32.");
        }

        Console.WriteLine("Το πρόγραμμα τερματίστηκε κανονικά.");
    }
}
